## Стек-ловушки (ВАЖНО — не сломать)
- **`DB.get(key)` фолбэчит на `ДАННЫЕ[key]`** (js/db.js) → дефолты в `ДАННЫЕ` должны быть ЧЕСТНЫМИ (health=null/пусто, rpgStats={}, achievements unlocked:false, dailyLog energy/focus=null, workouts=[]). Иначе фейк «переинъектируется».
- **Фейк синкается из облака** (KV `rpgStats`, `achievements`, `health_metrics`) и возвращается после локальной чистки. Решено в коде: rpgStats не тянем из KV (supabaseSync), RC/Сон гейтятся на `health.real`, ачивки — только с `earnedAt`. **Данные/схемы в облаке НЕ удаляем** (просьба пользователя). Delete-миграции не используем.
- **Приложение заинлайнено светлым текстом** (`color:rgba(232,237,245,..)`). Светлая тема (`html.light` в css/liquid-glass.css) переворачивает их атрибут-селекторами `[style*="232,237,245"]`. Новый UI — классами, не инлайн-цветом.
- **Chart.js + CSS-колонки (masonry) несовместимы** — радар улетал, раскладку «вело». Радар/энергия-канвасы убраны с дашборда. Не возвращать Chart.js внутрь `.dd-masonry`.
- **Headless-превью отдаёт width=0** → рендерит мобайл; десктоп-masonry в превью не проверить (проверять логику + реальный Safari). localStorage привязан к ПОРТУ (origin) — сидить/чистить на том же порту, что инспектируешь.
- **Tombstones:** `DB.markDeleted` пишет `lifeos_deleted`; pull в supabaseSync фильтрует эти id (tasks/people/content/idea_bank). Удалённое не воскресает.
- **Vercel Hobby лимит 12 функций** — `api/` на грани: 5 AI-endpoint объединены в `api/ai.js` (роутер) + rewrites в `vercel.json`. Не плодить новые `api/*.js`.
- **Бот:** модель из `config.life_model` (`/setmodel`, дефолт `gpt-5.5`). Парсинг — `парсJSONОтвет` (bot/model.js), `max_completion_tokens` 3000 (reasoning ест бюджет). Google-события — наивное мск-время + timeZone (без UTC-сдвига).
- **Env (Vercel):** нужен `HEALTH_SYNC_TOKEN`; есть `OPENAI_API_KEY`, `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`, Google creds, `NOTION_TOKEN`.
